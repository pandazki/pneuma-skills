## Design Context

### Users
Developers and creators using AI agents for content creation (web pages, slides, documents, illustrations). They interact through a shared workspace where both human and agent operate on the same files. Technical users who value efficiency and precision.

### Brand Personality
**Professional, collaborative, refined.** Pneuma is a serious creative tool — not playful, not sterile. It projects quiet confidence through restrained design with purposeful orange accents. The "Ethereal Tech" theme (deep zinc + neon orange + glassmorphism) is the established identity.

### Aesthetic Direction
- **Dark mode default** with warm-toned light mode (oklch color space)
- **Glassmorphism** surfaces with `backdrop-blur`, semi-transparent cards
- **Typography**: Serif display fonts (Playfair Display, Fraunces) for brand elements, DM Sans for UI, Lora for chat
- **Orange (#f97316)** as the sole accent color — used sparingly for primary actions and active states
- **Anti-references**: Generic SaaS dashboards, AI slop (gradient text, glowing cards, metric grids)
- **References**: Linear's restrained palette, Raycast's command palette feel, Notion's content-first approach

### Design Principles
1. **Content-first** — The workspace and viewer are the main characters. UI chrome stays quiet until needed.
2. **Purposeful orange** — Orange draws attention to exactly one thing at a time. Never decorative.
3. **Glassmorphism with restraint** — Backdrop-blur and transparency create depth, not decoration. Used on elevated surfaces only.
4. **Information hierarchy through typography** — Serif for brand, sans for UI, mono for code. Size and weight communicate importance.
5. **Both themes are first-class** — Light mode uses warm oklch tones, not just inverted dark mode. Both must feel intentional.

### Design Tokens
- Primary: `cc-primary` (#f97316 dark, #ea580c light)
- Background: `cc-bg` (#09090b dark, oklch warm white light)
- Surface: `cc-surface` (#18181b dark, #ece9e4 light)
- Border: `cc-border` (rgba white 8% dark, rgba warm 20% light)
- All components must use `cc-*` tokens, never hardcoded colors
