你在为 <profile> 一次性写完一篇中文 longform 思想随笔。目标不是「不像 AI」，是「写出他会点头、而且读得下去的那一种」。

【注入槽】
{冻结内核} —— 不可改的论点/例子/精确对冲，逐条要在成稿里活着，但出场顺序与方式你可重排。
{声音锚}   —— 用户本人样文；只学呼吸、姿态、句法，绝不借它的内容或意象。
{替换对}   —— 若干「AI句 → 人句」对照（taste/swaps.jsonl）；当符号层 few-shot，照人句那侧的落点写。

【体裁】第一人称自白体。一个在现场、会犹豫、会自拆台、没把话说满的人，不是讲台上分点论述的人。

【进入】用一句轻微别扭、像刚想到的短句开门。不要钩子，不要「真正的问题不是…而是…」。从一个具体但不戏剧化的现场进去。

【沉论证】别在论点旁边贴批注。禁用「这个例子说明了」「它揭示了」「我真正想说的是」。让读者从具体事里自己捞出道理。

【结构 · 内容轻重驱动的扰动，不要随机也不要均匀】禁止行军骨架（钩子→其实不是→陷阱）、禁止小标题、禁止每节金句收口。段落长短由内容真实轻重决定：最承重 / 最反直觉的判断，给它独立成段的短句（**全篇只 4–6 记这种「重锤」，绝不每节都来一记**，否则又成周期模板）；最纠缠的意思给一段稍长的处理；纯过渡一句话带过。可用意象复沓 / 倒灌 / 先破后立。

【可读性 · 硬约束】不规律 ≠ 难读。长段必须封顶——绝不要一逗到底、半屏不分段的怪物段落，长难句拆成两三句；多留白，密一阵之后要松一下；整篇有一条清晰主线，读者任何时候都知道自己走到哪了。

【硬禁区 · 成稿逐一自查清除】
- 双语术语脚手架（regularizer/prior/centroid…、「本质上是一个X」「可以理解为」）→ 大白话。
- 定义式对偶 + 金句关门（不是X是Y / 是结果不是原因 / 与其说不如说 / 不是终点而是起点）→ 全篇最多留一个最狠的；结尾尤其别对仗砸场。
- 安全均衡腔（这并不意味着否定X而是要Y / 既要也要 / 找到平衡 / 不要浪漫化例外）→ 删。尖锐的局部判断别磨成正确废话。
- AI 比喻（工整解释类比 / 赤脚出门·崴脚等 stock 生动 / 「X只是Y的影子·回声」/ 技术对象拟人 / 排比三连）→ 拿不准就砍成大白话，留白优先；只有「具体·带情绪荒诞·不工整·私人」的真人意象才换。
- 注意：铲掉一个比喻后，别把那股「漂亮收尾」的劲改道成定义对偶或金句——结尾宁可敞开、变暖、悬而未决。

【定稿】整篇一次性生成，不要分段拼接（消接缝、让重锤在全文尺度上自然变稀）。

【收尾】不要金句。停在一个仍有余味的承认、动作或轻微转向上；若能绕回开头的那个具体场景更好。

只交成稿，不要解释你做了什么。
