# 最高杠杆的偏离

我最近被一件很小的事烦到过。

我写了一段不太标准的代码，逻辑是对的，也不是为了显得聪明才绕路。只是那个地方有一点现场信息，写成最常见的样子反而会出问题。然后 AI 助手一遍遍把它改回大家最熟的写法。你改回来，我删掉；我再解释，它再换一种语气改回来。到最后我甚至有点怀疑自己，是不是我在某个地方太执拗了。

后来看到 Karpathy 提过一个类似的例子。他在 8 张 GPU 上同步梯度，没有用业界标准的 DDP，而是自己手写了一套。AI 反复把他往 DDP 那里拽，好像理解不了为什么有人在这个地方不用标准方案。问题当然不在于"他敢不一样"。要害是他知道一件标准方案不知道的现场信息。换一个不懂的人也不用 DDP，那不叫突破，只是把代码写差了。

这点很容易被讲歪。

我一开始也想把它归结成 AI 的问题。模型太保守，太爱平均，太像一个没在现场的人。可是想了一圈，又觉得这口锅给 AI 太便宜了。AI 在这里确实是一股把一切往"平均"上拽的力。它判断的是，你像不像过去那些被判对的东西，而不是在帮你做对。但这股力并不是 AI 发明的。

所谓最佳实践，本来就是过去无数次评判在很多场景上平均出来的东西。它通常有用，甚至大多数时候救命。问题在于它把很多细节抹平了。分布上对，落到你眼前这个具体场景，可能正好错。你手里那一点没写进文档的局部信息，平均值看不见。

以前这个张力藏在很多地方。藏在 code review 里，藏在框架文档里，藏在团队习惯里，也藏在自己手上的肌肉记忆里。AI 只是把它照亮了。它把那股"你为什么不像大家"的力量，变得很吵，很频繁，很难假装不存在。

但我也不太愿意顺着这个情绪往下滑。因为"偏离"这件事，太容易让人上头。

绝大多数偏离只是更差。

这句话说出来很扫兴，但我觉得必须放在前面。很多人看到平均有问题，就立刻觉得反平均大概更高级。其实不是。刻意唱反调和盲目随大流一样不自由，都是围着同一个平均转。你以为自己在逃离它，其实还在用它定义自己。

真正难的靶子不是"不一样"。真正难的是，在大家都搞错的地方，你恰好是对的。至于看起来不一样，只是事情发生之后留下来的样子。就像 Karpathy 那个例子，重点不是不用 DDP，重点是他知道为什么这里不用 DDP。如果把"没有用标准方案"单独拎出来学，就会学到最糟糕的部分。

我现在越来越觉得，进步大概总是长成一个类似的形状：先要有变化，而且这个变化不是乱抖，是有方向的。这个方向可能来自手感，来自一个还没被证明的假设，也可能来自别人没有的私有现场信息。然后还要有筛选。你得知道哪些变化留下，哪些变化丢掉。

只有变化，没有筛选，就是噪音。只有筛选，没有变化，就会一路收敛回现有的最佳实践。

后面这一半，正是 LLM 最擅长做的事。

这里也要小心，不能把话说成"LLM 不会创新"。它当然会产生训练集里逐字不存在的组合。它会重排，会嫁接，会把两个你没想到会放在一起的东西放在一起。它的限制不在于能不能新，而在于往哪个方向新。它更像是在见过的形状内部重新排列。你拿着一小块它没有的现场信息，刻意往外走一步，它就容易把这一步温和地推回去。

RLHF 还会让这件事更明显。很多奇怪的、冒险的、没那么讨喜的输出会被压下去，最后更稳，更顺，也更像大家都能接受的答案。用它来榨取已有空间里的收益，很合适。用它来探索，尤其是你已经知道自己必须离开中心一点的时候，它就可能有害。它不一定会粗暴反对你，它只是一次次把你的边角磨圆。

科学里也有类似的形状。先有猜想，再有反驳。演化里是变异和选择。单独一半都没什么用。没有变异，选择只能在旧东西里挑；没有选择，变异只是到处乱长。这个道理说起来朴素，但放回工作里就很难，因为我们太容易把"好的筛选器"当成中立的东西。

比如 eval、benchmark、metric。大家会觉得，解法可以激进一点，但评判标准总该客观一点吧。可负责筛选的东西，自己也是被冻结的最佳实践。它编码了上一轮共识里"什么算好"。你只在解法上偏离，却继续拿标准 metric 评判，最后还是会被拽回标准解。因为那个 metric 和标准解，本来就是同一套世界观的两面。

最高杠杆的偏离，常常发生在"评什么"上。

这也是为什么 benchmark 的故事总是让我有点着迷。拿 ImageNet 来说，不能把它讲成"ImageNet 创造了视觉能力"。能力不是 benchmark 凭空制造出来的。LeNet 在 1998 年就已经在那里了，卷积网络的路也不是 2012 年才从石头缝里冒出来。缺的是数据、算力，也缺一个让人相信这条路真的能交付的场面。

2010、2011 年，ImageNet 冠军还主要是 SVM 那一套。到了 2012 年，AlexNet 把错误率断崖式拉下来，领域几乎在一年内倒戈。ImageNet 做的事，不是创造能力，而是把一个已经潜在存在、却没人真信的能力变得可见，把注意力集中到那里。改 eval 改的是注意力，不是能力。只有当工具已经能交付，只是没人信的时候，改 eval 才推得动领域。

这里还有另一面。度量一旦变成目标，就会慢慢和现实脱钩。ImageNet 后来也饱和了，标签本身还有 6% 到 10% 的错误。它曾经让一个方向变得可见，也会在某个时候变成新的天花板。选择度量本身是最高杠杆动作，但这把刀不会因为你拿得很用力就变干净。

我脑子里常常把这件事想成一把梯子。最下面一层，是在给定 eval 里优化。绝大多数人都待在这里，LLM 也只能待在这里。你给它一个清楚的梯度，一个清楚的评分，它可以爬得很快，很稳，很像样。

再上一层，是选择到底优化哪个 eval。你开始意识到，问题不只是怎么拿高分，而是哪张卷子值得做。再往上，是重新定义什么才算有效的 eval，开出一条新的价值轴。到了那里，事情就不再像做题了。也正因为不像做题，很多假聪明和真突破会长得很像。

这时就绕不开 taste。

我以前对 taste 这个词有点抗拒。它很容易被讲成一种神秘气质，好像只要说"我有品味"，就不用解释自己为什么这么选。后来我慢慢觉得，得把它拆开看。生出变化那一端的手感和直觉，很多时候只是压缩过的经验。Simon 说直觉就是识别，棋手靠脑中成千上万的模式缓存下判断。这个说法我愿意接受。很多所谓手感，原则上是能还原的，只是还原起来很贵，很慢，很难写成文档。

它跟最佳实践其实同类。一个是公开的、低分辨率的经验压缩；一个是私有的、高分辨率的经验压缩。你在某个领域泡得足够久，见过足够多具体场面，就会在别人还没说完时觉得"不对"。这不是玄学，是识别。

可梯子最顶层不是识别。

最顶层是选择尺子本身。再多"过去什么算好"，也推不出"接下来什么应该算好"。这地方更接近 Kant 说的那种反思判断，也会碰到 Hume 那条"是"和"应该"之间的沟。你可以把过去的成功样本喂进去，抽出很多模式；但你仍然要问，下一把尺子为什么值得被立起来。

所以我会把结论放得窄一点：识别可还原，选择"什么值得被识别"不可还原。任何山 LLM 都能爬，只要你给它梯度和筛选器。它的问题不在爬山。它选不了哪座山值得爬。

说到这里，最危险的滑坡也就来了。因为一旦你说"我要重新定义什么算好"，下一句很容易变成"所以我现在已经成功了，只是你们看不懂"。这话听起来太熟了。每个失败者都可以这么说，每个骗子也可以这么说。更麻烦的是，很多真正后来被承认的人，在当下也只能说出差不多的话。

所以必须装一个刹车。

我以前很想把这个刹车叫作"现实"。听起来干净，结实，像最后的裁判。你可以定义你的标准，但现实会说话。可是这块地基并没有那么硬。现实落地的时候，还是要通过引用数、市场、benchmark、同行承认、用户行为、历史叙述这些东西。也就是说，它还是会变成下一轮 eval。

二十世纪科学哲学绕来绕去，几乎没人还站在"赤裸现实会干净说话"这边。Popper 说，哪个观察算反例，本身就有自由决定的成分，单个反常也杀不死一个理论。Kuhn 说，理论选择没有中立算法，观察总带着理论，最高标准常常是相关共同体的同意。Lakatos 更麻烦，他说一个研究纲领是进步还是退化，往往要回头看，判断还能被后来的历史推翻。Laudan 则干脆把那种想找充要标准来划界的努力拆得更散。

但这不等于什么都可以。

现实能否决，但不能加冕。反常有真实的负向约束力。火箭炸了就是炸了，用户不用就是不用，实验复现不出来就是复现不出来。你可以解释，可以等待，可以说当前框架不够好，但你不能假装这些都不存在。只是"这是不是 visionary"的正向加冕，永远要由一个更晚的、自己也会被推翻的评价框架给出。它可能迟到几十年，也可能反转。你一直是在对着一把会移动的尺子量自己。

这让人很不舒服。因为 visionary 和 crank 在动作上太像了。都偏离共识，都自定义标准，都说当前 eval 看不见自己做的东西。区别只在下游你控制不了的裁决怎么落。而当下你往往分不清自己是哪一个。更狠的是，所谓最终裁决也没有一个真的终局，只是后来的人暂时同意这样写。

我能接受的说法，大概只能到这里。顶尖的工作是一层层往上移动的 eval：先是在标准里做对，再到选择评什么，再到重新定义什么算好。连那个我原本以为能终审一切的裁判，也只是一个更晚的、还会被改写的 eval。

你能动的那一层，和你动不了的那一层，中间隔着 taste。taste 里真正不可还原的，不是手感，是你选择那把尺子的那一下。至于那一下后来会被叫作远见，还是被归进另一堆没人在意的误判里，我现在没有比以前更确定。只是下次 AI 又把我的代码改回标准写法时，我大概会先停几秒，问问自己：我手里那点现场信息，是真的存在，还是只是我不想承认自己写差了。
