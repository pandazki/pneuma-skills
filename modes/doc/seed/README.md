# Pneuma Doc

> 用自然语言描述你的想法，AI 为你编写和编辑文档 — 所有修改实时预览。

---

## 这是什么

你正在看到的是 **Pneuma Doc Mode** 的欢迎文档。右侧的聊天框里，AI 已经准备好了。

试试告诉它：

- "帮我写一篇关于 Rust 内存管理的技术博客"
- "把下面的要点扩写成完整段落"
- "给这个文档加上目录和更好的结构"

每一次编辑都会立即反映在左侧预览中 — 所见即所得。

---

## 核心能力

### 实时预览

AI 编辑 Markdown 文件 → 文件系统变更被监听 → WebSocket 推送到浏览器 → 预览立即刷新。整个过程不需要你做任何操作。

### 元素选择

点击左侧预览中的任意段落、标题、代码块或列表，被选中的元素会高亮显示，同时上下文信息自动注入到你的下一条消息中。AI 会知道你在看什么、选了什么。

**试试看** — 点击这个段落，然后在聊天框中说"把这段改成更简洁的版本"。

### 行内编辑

切换到 Edit 模式，你可以直接在预览中点击并编辑文本块。修改会自动保存到文件。

### GitHub Flavored Markdown

支持完整的 GFM 语法：

| 语法 | 效果 |
|------|------|
| `**粗体**` | **粗体** |
| `*斜体*` | *斜体* |
| `` `行内代码` `` | `行内代码` |
| `~~删除线~~` | ~~删除线~~ |

代码块支持语法高亮：

```typescript
// AI 可以帮你写代码块
function fibonacci(n: number): number {
  if (n <= 1) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
}
```

列表、任务列表也完全支持：

- [x] 实时预览
- [x] 元素选择
- [x] 行内编辑
- [ ] 你的下一个想法...

## 工作流提示

1. **从大纲开始** — 先让 AI 列出文档结构，再逐节填充
2. **选中后修改** — 点击不满意的段落，告诉 AI 怎么改
3. **多文件管理** — 你可以让 AI 创建多个 .md 文件，它们都会被监听和预览
4. **图片支持** — 在聊天框中粘贴或拖入图片，AI 可以看到并在文档中引用
