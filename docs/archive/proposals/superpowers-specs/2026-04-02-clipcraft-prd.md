# ClipCraft PRD — AI Video Production Mode

**Date:** 2026-04-02
**Status:** Draft — for discussion

## Problem

用 AI 做视频的流程是碎片化的：用户要在多个工具之间切换（ChatGPT 写脚本 → Midjourney 画图 → Kling 生视频 → CapCut 剪辑），每一步都需要手动搬运素材。Medeo 等 SaaS 解决了集成问题，但锁定在云端、模型不可选、成本不透明。

ClipCraft 要做的是：**在本地环境中，用 AI agent 编排整个视频生产流程，用户只需描述创意和做决策，agent 负责执行。**

## 用户是谁

- 内容创作者（短视频、自媒体）— 有创意但不会专业剪辑
- 开发者/设计师 — 想用 AI 快速出视频原型
- 团队 — 需要批量生产风格一致的视频内容

## 核心用户旅程

整个"做一个视频"的过程分为 **5 个阶段**，每个阶段有明确的输入、输出和 agent 角色：

### Phase 1: Brief — 描述你要什么

**用户做什么：** 用自然语言描述视频创意，或上传参考素材（图片、文档、URL）
**Agent 做什么：** 理解意图，确认关键参数（时长、风格、受众、宽高比）
**输出：** `project.json` — 项目定义

```
用户："做一个 30 秒的东京夜景短片，9:16 竖屏，lo-fi 风格，有旁白"
Agent："好的，我来规划一个 30s 的东京夜景短视频，9:16竖屏，lo-fi 氛围。"
```

**UI 需要：** 聊天就够了，不需要特殊 UI。

### Phase 2: Script — 分镜和脚本

**用户做什么：** 审阅 agent 生成的分镜，调整场景顺序/内容/旁白
**Agent 做什么：** 把创意拆解为场景序列，每个场景有：视觉描述、旁白文本、时长、转场
**输出：** `storyboard.json` — 带脚本的场景序列

```
Agent 输出：
Scene 1 (5s): 新宿街头霓虹，雨后反光 — "城市从不入睡..."
Scene 2 (4s): 小巷灯笼特写，散景背景 — "传统与现代在此相遇..."
Scene 3 (6s): 涩谷路口俯瞰，人流如织 — "从高处看，路口变成了艺术..."
Scene 4 (5s): 拉面碗热气腾腾 — "夜深了，总有一碗温暖等你..."
```

**UI 需要：**
- **Script Panel** — 场景列表，每个场景可编辑旁白文本
- **Scene Strip** — 可视化场景顺序和时长

这是 **创作的核心阶段**，用户在这里花时间最长。Agent 和用户来回迭代脚本直到满意。

### Phase 3: Generate — 素材生成

**用户做什么：** 确认后等待生成，对不满意的结果要求重新生成
**Agent 做什么：** 逐场景调用 MCP tools 生成视觉素材（图片/视频）
**输出：** 每个场景的 `assets/` 文件填充完毕

```
Agent: "开始生成第 1 个场景的视频..."
[调用 generate_image → 成功]
Agent: "第 1 个场景就绪。继续第 2 个..."
```

**UI 需要：**
- **生成进度** — 每个场景的状态（pending/generating/ready/error）
- **预览** — 生成完的素材实时显示
- **Assets Panel** — 管理素材（参考图、上传、删除）

### Phase 4: Assemble — 组装预览

**用户做什么：** 预览完整视频，调整节奏、转场、BGM
**Agent 做什么：** 组装时间轴，搜索/生成 BGM，调整时长和转场
**输出：** 完整可预览的 storyboard

**UI 需要：**
- **Video Preview** — 连续播放所有场景
- **Track Overview** — 字幕轨 + 视频轨 + BGM 轨
- **Caption Overlay** — 字幕预览

### Phase 5: Export — 导出

**用户做什么：** 选择格式和分辨率，导出
**Agent 做什么：** 生成 ffmpeg 脚本或调用渲染 API
**输出：** 最终视频文件

**UI 需要：** Export 按钮 + 格式选项

## 关键洞察

### 1. Phase 2 (Script) 是核心

Medeo 的 AI agent 最强的地方不是生成图片（这是 API 的事），而是**把模糊的创意变成结构化的分镜**。这一步决定了视频的质量。

ClipCraft 的 Script Panel 应该是**最重要的 UI 组件**：
- 不只是显示文本，而是一个结构化的脚本编辑器
- 每个场景卡片：视觉描述 + 旁白 + 时长 + 转场
- 可以拖拽调整顺序
- 可以内联编辑
- 可以一键让 agent 重写某个场景

### 2. 线性流程 vs 自由画布

当前的 viewer 试图把所有阶段同时展示（assets + script + preview + timeline），导致每个区域都很小。

**更好的方式：** 按阶段组织 UI，聚焦当前任务。

方案 A — **Tab-based phases:**
```
[Script] [Generate] [Preview] [Export]
```
每个 tab 是一个阶段的全屏 UI。Agent 引导用户在阶段间推进。

方案 B — **Adaptive layout:**
UI 根据当前阶段自动调整。写脚本时 Script Panel 占满，生成时显示进度，预览时全屏播放器。

方案 C — **保持当前布局但优化比例:**
左侧 Script/Assets 更宽，预览区更大，timeline 折叠。

### 3. 生成是异步的

视频生成要 30-120 秒，图片也要几秒。UI 需要优雅地处理等待状态，让用户在等待时能做其他事（编辑其他场景的脚本、调整已完成的素材）。

### 4. 迭代是逐场景的

用户很少重做整个视频，通常是"第 3 个场景不太好，换个角度"。UI 需要让单场景迭代非常方便。

## 重新思考 Viewer 布局

基于以上分析，建议的布局：

```
┌─────────────────────────────────────────────────────┐
│ [Script]  [Assets]                    [Caption: ON]  │ ← tab bar
├───────────────────────┬─────────────────────────────┤
│                       │                             │
│  Scene List           │     Video Preview           │
│  ┌─────────────────┐  │     (with caption overlay)  │
│  │ Sc1: 新宿街头    │  │                             │
│  │ 旁白: 城市从不... │  │     ┌─────────────────┐    │
│  │ 5s | crossfade  │  │     │                 │    │
│  ├─────────────────┤  │     │  [selected scene │    │
│  │ Sc2: 灯笼特写    │  │     │   preview]      │    │
│  │ 旁白: 传统与...   │  │     │                 │    │
│  │ 4s | crossfade  │  │     │  "城市从不入睡"   │    │
│  ├─────────────────┤  │     └─────────────────┘    │
│  │ Sc3: ⏳生成中    │  │                             │
│  │ 旁白: 从高处看... │  │     [▶] 00:12 / 00:20      │
│  │ 6s | fade       │  │     [16:9] [9:16] [1:1]     │
│  ├─────────────────┤  │                             │
│  │ Sc4: 拉面碗      │  │                             │
│  └─────────────────┘  │                             │
│  [+ Add Scene]        │                             │
├───────────────────────┴─────────────────────────────┤
│ Tt │ 城市从不... │ 传统与... │ 从高处看... │ 夜深了... │ │
│ 🎬 │ ████████  │ ████████ │  ░░░░░░░░  │ ████████ │ │
│ ♪  │ ═══════ Tokyo Midnight Lofi ════════════════  │ │
└─────────────────────────────────────────────────────┘
```

关键改进：
1. **Scene List 替代 Scene Strip** — 左侧纵向列表取代底部横向卡片，每个场景卡片信息更丰富
2. **场景卡片可编辑** — 直接编辑旁白、调整时长，不需要回到聊天
3. **Preview 更大** — 去掉横向场景条后预览区更大
4. **Timeline 精简** — 保持底部 track overview，但更紧凑
5. **Assets 作为 tab** — 不同时显示，按需切换

## 待讨论

1. **Tab-based vs 全部同时展示？** — 你觉得 Script/Assets 用 tab 切换合适，还是应该同时可见？
2. **Scene List 纵向 vs Scene Strip 横向？** — 纵向能显示更多信息（旁白、时长），横向更适合快速浏览缩略图
3. **场景内联编辑** — 直接在场景卡片里改旁白/时长，还是选中后在右侧面板改？
4. **Phase-based UI 切换** — 是否需要根据当前阶段（脚本/生成/预览）自动调整布局？
5. **BGM 怎么选？** — 现在是 stub，实际产品里 BGM 的选择和预览体验应该怎么设计？
