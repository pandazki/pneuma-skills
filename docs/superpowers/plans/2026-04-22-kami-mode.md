> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a new independent builtin mode `kami` to Pneuma — paper-canvas web design with warm parchment aesthetic adapted from [tw93/kami](https://github.com/tw93/kami) (MIT). Paper size locked at workspace creation via new `"select"` init-param type. Seeds three demo content sets from kami's README (tesla one-pager CN, musk resume EN, kaku portfolio CN) plus a blank starter.

**Architecture:** Fork webcraft's pattern cleanly — kami is a sibling mode, not an extension. Mode-package layout: `modes/kami/{manifest,domain,pneuma-mode}.ts` + `viewer/KamiPreview.tsx` + `seed/{_shared,tesla-one-pager,musk-resume,kaku-portfolio,blank}/` + `skill/SKILL.md` + `skill/references/*.md` + `NOTICE.md`. Paper size flows: launcher `<select>` → `deriveParams` → `.pneuma/config.json` → skill-installer `{{pageWidthMm}}` substitution → `seed/_shared/styles.css` CSS variables → viewer reads config source for the locked preset.

**Tech Stack:** TypeScript (strict), Bun runtime, React 19 + Vite 7, Hono 4.7, Zustand, chokidar. Same stack as the rest of Pneuma — no new dependencies.

**Reference spec:** `docs/superpowers/specs/2026-04-22-kami-mode-design.md` (commit `5bb61f3`).

```ts
/** Mode init parameter declaration — interactively prompted on first launch */
export interface InitParam {
  /** Parameter name, also used as template placeholder key (e.g. "slideWidth") */
  name: string;
  /** Display label for interactive prompt (e.g. "Slide width") */
  label: string;
  /** Additional description (e.g. "pixels") */
  description?: string;
  /** Parameter type. "select" requires `options` to be set. */
  type: "number" | "string" | "select";
  /** Required when type === "select". List of string choices shown to the user. */
  options?: string[];
  /** Default value. For "select", must be one of `options`. */
  defaultValue: number | string;
  /** Mark as sensitive value (API keys, etc.), cleared during snapshot packaging */
  sensitive?: boolean;
}
```

```ts
  for (const param of initParams) {
    const effectiveDefault = defaultOverrides?.[param.name] ?? String(param.defaultValue);
    const suffix = param.description ? ` (${param.description})` : "";

    if (param.type === "select") {
      if (!param.options || param.options.length === 0) {
        throw new Error(`InitParam "${param.name}" has type "select" but no options`);
      }
      const answer = await p.select({
        message: `${param.label}${suffix}`,
        options: param.options.map((o) => ({ value: o, label: o })),
        initialValue: effectiveDefault,
      });
      if (p.isCancel(answer)) {
        p.cancel("Cancelled.");
        process.exit(0);
      }
      params[param.name] = String(answer);
      continue;
    }

    const answer = await p.text({
      message: `${param.label}${suffix}`,
      placeholder: effectiveDefault,
      defaultValue: effectiveDefault,
    });
    if (p.isCancel(answer)) {
      p.cancel("Cancelled.");
      process.exit(0);
    }
    if (answer === "" || answer === String(param.defaultValue)) {
      params[param.name] = param.defaultValue;
    } else if (param.type === "number") {
      const num = Number(answer);
      params[param.name] = isNaN(num) ? param.defaultValue : num;
    } else {
      params[param.name] = answer;
    }
  }
```

```markdown
## Aesthetic rules (kami adapted)

| Element | Rule |
|---|---|
| Canvas | `#f5f4ed` parchment. Never pure white. |
| Accent | Ink blue `#1B365D` only. No second chromatic hue. |
| Neutrals | Warm-toned (yellow-brown undertone). No cool blue-grays. |
| Serif | `TsangerJinKai02` (CN) / `Newsreader` (EN). Weight locked 500. Never bold. |
| Line-height | Titles 1.1–1.3. Body 1.5–1.55. Never 1.6+. |
| Shadows | Ring or whisper only. No hard drop shadows. No gradients. |
| Tags | Solid hex backgrounds only — rgba() can break in print. |

If the user writes in Chinese, prefer the CN demos and Chinese typography
fallbacks. If they write in English, prefer the EN demos and Newsreader.
```

```tsx
}
```
